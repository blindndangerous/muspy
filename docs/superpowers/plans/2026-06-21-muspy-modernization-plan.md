# Muspy modernization foundation implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the repo root with a verified modern Django 6 foundation while preserving the legacy app under `legacy/`.

**Architecture:** This plan creates the project foundation only. It moves legacy code out of the root, adds a new Django project named `config`, creates the initial `releasewatch` app, configures `uv`, PostgreSQL 18, Podman/Docker Compose, CI, security checks, and a guarded dev admin bootstrap. Later plans will add domain models, upstream clients, release sync, notifications, feeds, and UI.

**Tech stack:** Python 3.14, Django 6, PostgreSQL 18, `uv`, `pytest`, `pytest-django`, `coverage`, `ruff`, `bandit`, Podman Compose or Docker Compose.

---

## Scope check

The approved design covers several subsystems: foundation, domain model, upstream clients, release sync, notifications, feeds, UI, and hardening. One monolithic implementation plan would be too large to execute safely. This file is the first implementation plan and covers only the foundation/scaffold phase. It must leave a working, testable app that later plans can build on.

## File structure after this plan

- `legacy/`: existing Python 2/Django 1.3 Muspy app, moved intact for reference.
- `README.md`: modern project entry point for GitHub users.
- `docs/`: architecture, development, deployment, security, handoff, and superpowers docs.
- `pyproject.toml`: `uv` project metadata, dependencies, pytest, coverage, ruff, and bandit config.
- `uv.lock`: locked dependency graph generated by `uv lock`.
- `.python-version`: requested Python version for `uv`.

- `tests/`: foundation tests.
- `compose.yml`: Podman/Docker Compose services.
- `Containerfile`: OCI image build.
- `.github/workflows/ci.yml`: CI checks.

```python
import os

from django.conf import settings
from django.contrib.auth import get_user_model
from django.core.management.base import BaseCommand, CommandError


class Command(BaseCommand):
    help = "Create or update a local development admin account."

    def handle(self, *args, **options):
        allow_override = os.environ.get("ALLOW_DEV_ADMIN_BOOTSTRAP", "").lower() in {
            "1",
            "true",
            "yes",
            "on",
        }
        if not settings.DEBUG and not allow_override:
            raise CommandError("ensure_dev_admin refuses to run without DEBUG or override")

        username = os.environ.get("DEV_ADMIN_USERNAME", "admin")
        email = os.environ.get("DEV_ADMIN_EMAIL", "admin@example.test")
        password = os.environ.get("DEV_ADMIN_PASSWORD")
        if not password:
            raise CommandError("DEV_ADMIN_PASSWORD is required")

        user_model = get_user_model()
        user, _created = user_model.objects.get_or_create(
            username=username,
            defaults={"email": email, "is_staff": True, "is_superuser": True},
        )
        user.email = email
        user.is_staff = True
        user.is_superuser = True
        user.set_password(password)
        user.save()

        self.stdout.write(self.style.SUCCESS(f"Development admin ready: {username}"))
```

# Development

## Requirements

- `uv`
- PostgreSQL 18, or Podman/Docker Compose
- Python 3.14 managed by `uv`

## Local setup

Copy the environment template:

```powershell
Copy-Item .env.example .env
```

Install dependencies:

```powershell
uv sync --locked --all-extras --dev
```

Run migrations:

```powershell
uv run python manage.py migrate
```

Create a local admin:

```powershell
uv run python manage.py ensure_dev_admin
```

Run the server:

```powershell
uv run python manage.py runserver
```

## Tests

Run targeted tests first while developing:

```powershell
uv run pytest tests/test_project_smoke.py -q
```

Run the full suite:

```powershell
uv run coverage run -m pytest
uv run coverage report
```

Run lint and security checks:

```powershell
uv run ruff check .
uv run bandit -r config releasewatch
uv run python manage.py check
```

## TDD standard

Write a failing test before production code for each new behavior. Run it and confirm the failure is caused by missing behavior, then write the smallest implementation that passes.
````

```powershell

uv run python manage.py migrate
uv run python manage.py check --deploy

`ensure_dev_admin` is for local/dev bootstrap. It refuses to run in production unless `ALLOW_DEV_ADMIN_BOOTSTRAP=1` is set. Do not use a shared or committed password.

# Security

## Baseline

- Secrets come from environment variables.
- Production must run with `DEBUG=0`.
- Mutating forms use CSRF protection.
- Feed and iCal URLs use revocable tokens.
- User-owned data must be checked by owner in every view and query.
- Email notifications require verified email addresses.
- Account deletion must remove or anonymize user-owned data.

## Checks

Run:

```powershell
uv run ruff check .
uv run bandit -r config releasewatch
uv run python manage.py check --deploy
uv run coverage run -m pytest
uv run coverage report
```
````
